Semantic Blueprint

Hover over any element in the wireframe layout to understand its precise semantic role, SEO purpose, and rendering rules!

<header>

Declares introductory site content, brand identity logos, search tools, and main navigational titles. Placed at the top of pages or separate articles.

<nav>

Groups major navigation menu blocks. Screen readers use this tag to allow users to skip directly to site-wide menu lists.

<main>

Specifies the unique, core layout of the current document page. There must only be one visible <main> per document.

<section>

A thematic grouping of information, typically starting with an explicit heading tag (H1-H6). Ex: a chapter list, features panel, or FAQ list.

<article>

Represents a self-contained, independent composition distributable outside the page context. Ex: a forum post, blog article, product review, or interactive widget card.

<aside>

Identifies secondary layout sidebars, glossary side panels, advertising containers, or quote summaries that are tangentially relevant to primary article content.

<footer>

Defines structural ending details for its parent page or block. Contains copyright notes, back-to-top links, sitemap arrays, and contact data.

The CSS Box Model Decoded

All elements within a document are presented using a rectangular layout box. Knowledge about the sum total of margins, borders, padding, and contents forms the very basis of web design.

MARGIN
BORDER
PADDING
CONTENT width × height

Interactive Box Model

Point your mouse at the different layers of the model container (i.e., Content, Padding, Border, or Margin) and discover how size and layout properties work!

Always:

Use box-sizing: border-box to make padding and borders fit inside the element's defined width, rather than expanding it outwards!

Margin (Outer Spacing)

margin creates transparent space *outside* the element's border. It separates adjacent elements. 

margin: 24px; /* All sides */
margin-top: 10px; /* Specific */
margin: 0 auto; /* Center block */

Border (Boundary Edge)

border surrounds the padding and content. You can style its thickness, texture style, and color. 

border: 2px solid var(--primary);
border-radius: 8px; /* Curved edges */
border-bottom: 1px dashed red;

Padding (Internal Breathing Room)

padding creates transparent breathing space *inside* the element, separating content from the border boundary. 

padding: 16px; /* Uniform sides */
padding: 12px 24px; /* Y & X axis */
padding-left: 5px; /* Left edge */

Content (Core Data Area)

content represents the actual text strings, image tags, or child elements nested inside. 

width: 300px; /* Layout width */
height: auto; /* Fit to text size */
box-sizing: border-box; /* Safe size */

CSS Target Mechanics

Hover any CSS selector listed in the left-hand column to observe how well it selects HTML elements, along with the syntax of the selector and its practical examples.

Class Selectors (.class)

Selects an element with only one match of the ID attribute. ID selectors are highly specific and should never be repeated. 

.highlight-badge {
  background-color: gold;
}

ID Selectors (#id)

Selects HTML elements that carry specific attribute values. Commonly used to style form controls. 

#main-navigation-header {
  position: sticky;
}

Attribute Selectors ([name="val"])

Applies visual properties depending on the transient states of interaction with an element (such as hover, focus, selection, etc.). 

input[disabled] {
  opacity: 0.5;
  cursor: not-allowed;
}

Pseudo-classes (:state)

Selects a target element B immediately adjacent to the sibling element A. Helpful for controlling margins. 

.ctrl-btn:hover {
  transform: translateY(-2px);
}

Adjacency Combinators (A + B)

Matches a target element B that is positioned immediately adjacent to a sibling element A. Useful for margin controls. 

h2 + p {
  margin-top: 0.5rem;
}

Understanding @keyframes Syntax

The @keyframes rule tells the browser rendering engine how to transition styles through time percentages:

The transition smoothly interpolates from initial structure (0%) to final state (100%). 

/* Sinusoidal Float Keyframe */
@keyframes float {
  0%, 100% {
    transform: translateY(0px);
  }
  50% {
    transform: translateY(-10px);
  }
}
